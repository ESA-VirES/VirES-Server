Changelog

Release of Service v3.17.0 (2025-12-20)

Features:

  • Support for EFIx_LP_1B baseline 07
    This release implements support for the new baseline of the Swarm OPER EFIx_LP_1B products.

Breaking changes:

  • EFIx_LP_1B baseline 07
    The new Swarm EFIx_LP_1B baseline 07 defines new and changes existing product variables.

Release of Service v3.16.0 (2025-12-10)

Features:

  • Web client visualization of Swarm L2 MIT and PPI products.
    It is now possible to visualize the data from the MITx_LP_2F, MITxTEC_2F, and PPIxFAC_2F products in the VirES web client.
  • CHAOS-8 MIO model
    The ionospheric component of the CHAOS-8 magetic model is now available.
  • Interpolation of Swarm 50Hz HR model values from 1Hz values
    As a performance optimization, the magnetic model values for the Swarm MAGx_HR_1B times and locations served by the VirES APIs are now interpolated from the magnetic model values at the MAGx_LR_1B 1Hz times and locations (see VirES Python client documentation).
  • MAGx_HR_1B CHAOS model and residuals served via HAPI
    The VirES HAPI endpoint serves CHAOS magnetic model and residuals for the high-resolution Swarm MAGx_HR_1B magnetic field measurements.
  • New API for calculation of geo-magnetic models at user-provided times and locations
    The VirES API allows calculation of model values at user-provided times and location. This API will be soon exposed via the VirES VirES Python client.

Breaking changes:

  • VirES API strict enforcement of the correct request content type
    The new VirES API for calculation of model values at user-provided times and locations requires use of special HTTP/POST requests with attached binary content. In order to support these requests, strict enforcement of the request content type had to be implemented. Unfortunately, as result of this change, older versions of the Python client (< 0.12.2) may not longer work.
    If you see an error like Unsupported application/x-www-form-urlencoded request format!, please, upgrade to the latest version of the VirES Python client.

Release of Service v3.15.2 (2025-04-28)

Features:

  • New Swarm L2 ULF and PC1 products
    The Swarm magnetic field ultra low frequency wave characterisation ULFxMAG_2F and PC1xMAG_2F, created by the Swarm DISC ULF project, have been added to VirES and are accessible via the VirES Python client and HAPI.
  • New Swarm L2 NEGIX and TEGIX products
    The Swarm electron density (NEGIX) and total electron content (TEGIX) gradient ionosphere indices, created by the Swarm DISC MIGRAS project, have been added to VirES and are accessible via the VirES Python client and HAPI.
  • New CHAMP, GRACE, and GRACE-FO electron density and TEC products
    The CHAMP, GRACE, and GRACE-FO dual-spacecraft average electron density total electron content (TEC) product, created by the Swarm DISC TIRO project, have been added to VirES and are accessible via the VirES Python client and HAPI.

Release of Service v3.12.0 (2023-09-04)

Features:

  • Caching of selected slow magnetic models
    To speed-up the calculation of the most frequently use magnetic models (CHAOS, Swarm-CI) selected slow-to-calculate MLI (CHAOS-Static, SW_MLI_SHA_2C) and MIO (SW_MIO_SHA_2C) magnetic modes are pre-calculated for the spacecraft locations are pulled from the cache rather that calculated on the fly.
  • MAG products served via HAPI provide reference magnetic model
    All MAG products served by VirES via the HAPI except the Swarm MG HR provide reference values calculated from the complete CHAOS model and the residuals when subtracted from the measurements.

Release of Service v3.10.0 (2022-10-22)

Features:

  • Extended data filtering API
    The VirES server API has been modified to allow new types of data filters. It is now possible to filter data by various equality and inequality predicates and masked bit flags. The new filters API can be fully exploited via the VirES Python client.
  • Bit-flags filters in web client
    The VirES web client now allows data filtering by binary quality and calibration flags. Thanks to the new server API, the same filters can be applied to the downloaded data.
  • Plotting error-bars
    The Analytical Viewer is now capable of plotting uncertainty as error bars.
  • New EFIxIDM products
    The new ion drift, density and effective mass products (EFIxIDM) prepared by the SLIDEM project have been added to VirES. The data is accessible via both API and web-client.
  • EFIxTIE and EFIxTCT products accessible via web-client
    The data from the Thermal Ion Imager cross track flow (EFIxTCT, 2 Hz) and ion temperature estimate (EFIxTIE), added to VirES in the previous release, have been made accessible via the web client.
  • New GOCE calibrated magnetic field measurements
    The calibrated magnetic field measurements from the GOCE mission has been added to VirES and are accessible via the API.

    Release of Service v3.0.0 (2019-05-10)

    Features:

    • Full source products/models traceability
      Information about the source products/models and their versions is now available for both the downloaded data and data displayed in the client.
    • Improved composed model GUI
      The client-side handling of the composed models has been thoroughly reviewed. The client GUI now shows information about the source models and the composed model expressions are now consistently shown throughout different parts of the web client.
    • Enhanced field-lines
      The displayed field-lines can be now selected by clicking. Information about the apex and ground intersections is displayed for a selected field-line.
    • New cyclic color-maps
      Two new cyclic color-maps (twilight and twilight-shifted) have been added to be used with cyclic variables such as latitude or MLT.

      A possibility to upload own data in the CDF file and compare them with the Swarm products has been added. The supported file format is identical the format of the downloaded data and most of the Swarm products (e.g., MAGx_LR).
    • Machine-to-machine (M2M) interface
      The VirES server M2M interface has been opened for registered VirES users. This interface has been so far accessible only indirectly trough web client. Now it can used either directly via the low-level API (based on the OGC WPS protocol) or via an alternative VirES client (e.g. VirES Python Client.
    • Token-based authentication for the M2M interface
      The access to the VirES server M2M interface is granted only for authenticated requests. A new option to use the token-based authentication has been added to VirES. The access tokens can be generated and revoked by the users themselves via the user settings of the web client.

    Release of Service v2.1 (2017-10-20)

    Features:

    • Alpha and Charlie satellite residuals
      The magnetic residual between satellites Alpha and Charlie in the NEC-frame have been added as parameters. The residuals can be accessed together with magnetic data from Swarm Alpha.
    • Orbit number
      The orbit number has been added as a parameter counting the number of Equator crossings of the ascending node since the Swarm mission start.
    • Spacecraft as product parameter
      A new parameter "Spacecraft" has been introduced to downloaded data to easily identify which satellite downloaded data originates from within a product.
    • Solar position
      Several parameters to determine the position of the sun have been added as filters:
      • SunDeclination, SunRightAscension and SunHourAngle defining Sun position in the Equatorial Coordinate System
      • SunZenithAngle, SunAzimuthAngle defining Sun position in the local Horizontal Coordinate System
      See "Equatorial coordinate system" and "Horizontal coordinate system".
    • All new parameters described above can be downloaded as part of a product by selecting them with the "Custom download parameters" in the Download panel.
    • Cursor coordinates on globe
      On hover of the globe in the Globe View, latitude and longitude for that point will be displayed in the top-left corner.
    • Extended Download panel functionality
      Parameter filters can be added directly in the Download panel by selecting "Add filter" in the Download panel. A Timestep filter enabling subsampling of the product. Subsampling can be enabled by selecting "Custom time subsampling" and specifying subsample interval in the Download panel. Subsampling will be applied to each daily product independently.
    • Product names in Layers
      Several data selections have been renamed:
      • "Electric Field Data" changed to "Equatorial electric field (EEF)"
      • "Electron data" changed to "Total electron content (TEC)"
      • "Currents data" changed to "Electric current data (FAC)"
      • "Non S/C" changed to "AC" (currently only includes combined S/C FAC data)
    • Model difference
      The difference between two models at any coefficient range can be displayed on the globe. This functionality can be accessed under settings of any of the geomagnetic models available in the Layers panel.
    • CHAOS-6
      Geomagnetic field model CHAOS-5 has been replaced by CHAOS-6 in the Layers panel.
    • Cartographic layer update
      "Blue Marble" layer previously available in the Layers panel has been updated with a "S2 Cloudless" mosaic, which has higher resolution and is based on Sentinel-2 data.

    Release of Service v2.0 (2017-03-20)

    Features:

    • Asynchronous processing
      It is now possible to download filtered Swarm data from an arbitrarily large time span to one file. This can be done by defining your filters in the analytics view (in addition to area selection on the globe view) for a smaller time span. Once satisfied with your selection you can click on download. In the Download Manager which then appears you can set the time window of interest, then click on "Process & create link". The processing is then done in the background, the progress can be seen in the download panel. Once finished a download link is provided there for you to get the results. Download limit will be set to the equivalent of 50 Magnetic low rate products.
    • Ionospheric total electron content data
      The data from TECxTMS_2F has been added to VirES with the possibility to view the Absolute slant TEC, Relative slant TEC and the root mean square error of relative slant TEC on the globe. A new visualization was created for this product to display the TEC parameter value for each of the paths to nearby GPS satellites simultaneously. The values for a timestamp are shown one above the other, allowing one to quickly see how many satellites were visible and what the values were for each timestamp shown.
    • Field aligned current
      The data from FACxTMS_2F (single S/C soluton) and FAC_TMS_2F (combined S/C solution) has been added to VirES with the possibility to view the Field-aligned current (FAC) and Ionospheric radial current (IRC) on the globe and in the Analytics View. The combined S/C solution data can be found by selecting "Non S/C" and "Currents data (FAC)" in the Layers widget.
    • Dayside equatorial electric field
      The data from EEFxTMS_2F has been added to VirES with the possibility to view the Estimate of the equatorial electric field (EEF) on the globe and in the Analytics View.
    • Local evaluation of custom model
      Providing a more detailed rendering, one can restrict the evaluation of a custom uploaded spherical harmonics model to an area of interest with the Area Selection tool.
    • Updated backplane map
      The new version of the backplane map offers the following improvements:
      • Digital Elevation Model (DEM) data merged from SRTM, ASTER and regional/local data like EU-DEM
      • New shaded relief with better generalization and geotopographical features
      • Improved generalization of coastlines and land polygons
      • Updated OpenStreetMaps data
      • Removed data gap within Antarctica

    Release of Service v1.2 (2016-12-21)

    Features:

    • Resume session
      Upon termination of your session information about the view, time window, area, data and filter selections and all corresponding settings will be stored for your convenience. Upon restart of the service, your session will be resumed where you left off. The session state information is stored locally with your internet browser, unless VirES is accessed using private browsing.
    • Jump to date
      To quickly access data from any part of the Swarm mission, press on the button in the top right corner of the timeslider slider. This will allow you to display data from a date of your choice. The time selection will be on a corresponding time of day as your previous time selection. If the previous time selection passes over midnight, the selection will be clipped to end at midnight
    • Set detailed download time filter
      In addition to filtering Swarm products by a wide range of parameters when downloading, you can select "Custom time selection" in the Download Manager. This will make it possible to define a precise start and end time for the downloaded product. Other filters shown in the Download Manager will still apply to the data
    • New overlay layers
      Three new overlays are available from the layers panel:
      • Coastlines and Countries: Black coastlines with light grey country borders
      • Graticules: Grid lines for geographic longitude/latitude
      • Magnetic graticules: Grid lines for magnetic longitude/latitude
    • Additional colour scales
      When selecting Swarm products or models to display on the globe, the colour representation of the data can be set in the respective settings widgets found in the Layers widget. It is now possible to choose between 27 scales including ones such as viridis, plasma and portland. The names of custom1 and custom2 colour scales have been changed to diverging_1 and diverging_2 to better represent the corresponding style.

    Release of Service v1.1 (2016-09-26)

    Features:

    • Plasma products
      The data from EFIX_PL_1B have been added to VirES with the possibility to view the plasma density and temperature, in addition to the spacecraft velocity and potential both on the globe and in the Analytics View. The full set of parameters are accessible on download
    • Ionospheric bubble product
      The data from IBI_TMS_2F have been added to VirES with the possibility to view the bubble probability on the globe and in the Analytics View. Positive Bubble index detections are shown in the time slider. The full set of parameters are accessible on download
    • Layers interface changes
      The Layers widget has changed separating out satellite specific products. Selecting a satellite specific product is now independent of selecting what satellites to view data from. Selection of satellite is done using separate checkboxes in the Layers widget. This change was introduced to make the Layers widget more compact after adding more products.
    • Customize download fields
      When downloading data, in addition to customizing the exact filter values and format, it is possible to specify what parameters to include by ticking off on "Custom download parameters", then either select the data from a dropdown menu or type it in. This customization allows for parameters from multiple Swarm products.
    • Label with colorscale information directly integrated into globe visualization
      Now when using the save function the currently configured color scale and ranges are saved also into the image. It is possible to toggle the visualization of the label in the settings panel for each Layer using the "Legend" tickbox.
    • Dynamic range calculation for models
      When changing model parameters (for SIFM, IGRF12, CHAOS-5 and Custom Model), such as the coefficient range or height, the colorscale range is calculated and adjusted automatically. It is still possible to manually adjust the range with both textfields available in the settings panel. This change was introduced as it became clear that finding the correct range after changing model parameters was cumbersome for users and we wanted to facilitate the process.
    • Single axis interaction
      When exploring the scatterplot, one can scroll when hovering over an axis to zoom along one axis while keeping the other axes ranges fixed. Similarily one can drag on an axis to pan the plot. These features apply to both the x-axis and the y-axes and complement the zooming and panning one can perform on the scatterplot itself to zoom along all axes.
    • Zoom limit removed
      The zoom limit previously applied to the scatterplot is now removed. There are now no restrictions to how far you can zoom in or out. A reset button is added to enable the user to return to the initial zoom level and position at one's convenience.
    • Customizable filter list
      The data filter histograms found in the Analytics View can be removed individually, and new histograms for any other available parameter can be added by selecting it in the histogram "Add" dropdown menu. Any active filters will be removed when its corresponding histogram is removed. In addition the filters can be hidden to allow for more space for the scatterplot.
    • Secondary y-axis
      When selecting multiple parameters a second y-axis will appear. When selecting two parameters this means that each parameter has its own axis. If more than two parameters are selected, they will be grouped according to their range, such that the parameters with the most similar value ranges are grouped together and share an axis.
    • Scatterplot units
      The units are now displayed in a parenthesis next to the axis parameter labels in the scatterplot.
    • Extended time selection
      The time selection limit when selecting a time window from the time slider has been extended from 15 days to 30 days. The limit when downloading products has not changed and is still set at the equivalent of 5 unfiltered MAGX_LR products.
